# Elehant Water для Home Assistant

Неофициальная пользовательская интеграция Home Assistant для локального чтения собственных счетчиков, продаваемых под торговой маркой Elehant, из Bluetooth LE advertising-пакетов.

Elehant является торговой маркой счетчиков. Этот проект не является официальным продуктом Elehant, не связан с производителем и не одобрен им. Интеграция предназначена для чтения собственных счетчиков в Home Assistant и не заменяет штатные средства, официальную передачу показаний, поверку или иные обязательные метрологические процедуры.

Этот репозиторий является форком и глубоким рефакторингом исходного проекта:

- исходный проект: https://github.com/raxers/elehant_water
- автор исходного проекта: raxErs / raxers

Спасибо автору исходного проекта за первоначальную реализацию поддержки BLE-пакетов счетчиков Elehant и первую рабочую интеграцию для Home Assistant. Текущий форк продолжает эту работу и сохраняет лицензию GNU GPLv3.

## О Форке

Форк переработан для совместимости с актуальными версиями Home Assistant, стандартизации поведения в UI и дальнейшего развития проекта.

Минимальная протестированная версия:

- **Home Assistant Core 2026.5.x**

Основные изменения:

- используется штатный Bluetooth API Home Assistant;
- конфигурация хранится в config entry;
- поддерживается автоматический импорт старой YAML-конфигурации;
- сохраняются legacy `unique_id` сущностей при миграции;
- счетчик идентифицируется canonical 24-bit Elehant serial number;
- объемные сенсоры получают `state_class=total_increasing`;
- температурные сенсоры получают `state_class=measurement`;
- доступность сенсоров определяется через стандартный `available` механизм Home Assistant;
- по умолчанию при миграции задается timeout доступности 60 минут;
- добавлены подробные diagnostics для отладки BLE, identity matching и stale-состояний.

## Поддерживаемые Устройства

Сейчас интеграция ориентирована на совместимые водяные счетчики Elehant:

- однотарифные пакеты с BLE-префиксом адреса `B0:01:02`;
- двухтарифные пакеты с BLE-префиксами `B0:03:02` и `B0:04:02` в пределах поведения, унаследованного от исходного проекта.

Газовые и другие счетчики Elehant пока не заявлены как функционально поддерживаемые без реальных устройств или надежных BLE-сэмплов.

## Установка Через HACS

1. Сделайте резервную копию конфигурации Home Assistant.
2. Откройте Home Assistant.
3. Перейдите в **HACS**.
4. Откройте меню с тремя точками.
5. Выберите **Custom repositories**.
6. Добавьте URL этого репозитория.
7. В категории выберите **Integration**.
8. Установите интеграцию **Elehant Water**.
9. Перезапустите Home Assistant.

## Ручная Установка

Скопируйте папку:

```text
custom_components/elehant_water
```

в каталог конфигурации Home Assistant:

```text
<config>/custom_components/elehant_water
```

После копирования перезапустите Home Assistant.

## Миграция YAML

Если у вас уже есть YAML-конфигурация старой интеграции, оставьте ее в `configuration.yaml` на первый запуск после установки форка.

Пример:

```yaml
sensor:
  - platform: elehant_water
    scan_duration: 10
    scan_interval: 600
    measurement_water: m3
    measurement_gas: m3
    devices:
      - id: 18674
        type: water
        water_type: hot
        name: "Вода Горячая"
        name_temp: "Вода Горячая температура"
      - id: 92728
        type: water
        water_type: cold
        name: "Вода Холодная"
        name_temp: "Вода Холодная температура"
```

Порядок миграции:

1. Установите интеграцию через HACS custom repository или вручную.
2. Оставьте старый YAML-блок `sensor: - platform: elehant_water` в `configuration.yaml`.
3. Перезапустите Home Assistant.
4. Интеграция импортирует YAML в config entry.
5. Проверьте, что сущности появились и сохранили ожидаемые `entity_id` и `unique_id`.
6. После успешной проверки удалите старый YAML-блок.
7. Перезапустите Home Assistant еще раз.

При миграции:

- короткие старые ID трактуются как тот же 24-битный номер Elehant с опущенными ведущими нулями;
- channel-level `legacy_id` сохраняется для совместимости `unique_id`;
- `availability_timeout_minutes` явно устанавливается в `60`;
- YAML используется только как путь миграции, не как основной формат настройки.

## Настройка В UI

После установки откройте:

```text
Settings -> Devices & services -> Elehant Water -> Configure
```

В options flow доступны:

- редактирование счетчиков;
- импорт найденной YAML-конфигурации;
- расширенный JSON-редактор для отладки.

Discovery flow использует canonical 24-bit meter ID и предлагает тот же timeout доступности, что и YAML-миграция.

## Bluetooth

Счетчики передают данные в non-connectable BLE advertisements. Интеграция использует штатный Bluetooth-стек Home Assistant с `connectable=False`.

Перед настройкой убедитесь, что:

- локальный Bluetooth-адаптер виден Home Assistant;
- или настроен Bluetooth Proxy;
- штатная интеграция Bluetooth не сообщает об ошибках.

## Troubleshooting

- Если интеграция не видит счетчики, проверьте наличие активного Bluetooth scanner или Bluetooth Proxy.
- Если HACS не видит свежий commit, обновите custom repository в HACS и перезапустите Home Assistant.
- Если после миграции нет сущностей, откройте options flow и используйте импорт YAML вручную.
- Если счетчик долго не обновлялся, сенсоры станут unavailable после настроенного timeout.
- Diagnostics в Home Assistant содержит подробные данные для отладки identity matching, RSSI, last seen и parser statistics.
- В HA Repairs появляются предупреждения о недоступном Bluetooth scanner, поврежденной конфигурации, дублирующихся ID и счетчиках, которые не были обнаружены после запуска.
- Для каждого канала создаются diagnostic-сущности last seen, RSSI, packet count и matched by. Они отключены по умолчанию и могут быть включены в настройках сущностей Home Assistant.

## Ограничения

- Реальные two-tariff BLE samples пока ограничены; расширение поведения двухтарифного парсера без сэмплов не выполняется.
- Поддержка газовых и других счетчиков Elehant вынесена в следующий цикл улучшений.
- Строгие Bluetooth manifest matchers не используются без надежного подтверждения manufacturer id/prefix.
