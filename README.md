# Неофициальная интеграция Home Assistant для счетчиков Elehant

Неофициальная пользовательская интеграция Home Assistant для локального чтения собственных счетчиков, продаваемых под маркой Elehant, из Bluetooth LE advertising-пакетов.

Elehant является торговой маркой соответствующих счетчиков. Этот проект не является официальным продуктом Elehant, не связан с производителем и не одобрен им.

Интеграция предназначена для локального чтения собственных счетчиков в Home Assistant. Она не заменяет штатные средства считывания, официальные процедуры передачи показаний, требования поставщиков коммунальных услуг, поверку или иные обязательные метрологические процедуры.

Этот репозиторий является форком и глубоким рефакторингом исходного проекта:

- исходный проект: https://github.com/raxers/elehant_water
- автор исходного проекта: raxErs / raxers

Спасибо автору исходного проекта за первоначальную реализацию поддержки BLE-пакетов счетчиков Elehant и первую рабочую интеграцию для Home Assistant. Текущий форк продолжает эту работу и сохраняет лицензию GNU GPLv3.

## О Форке

Этот форк - глубокий рефакторинг исходной YAML-автоматизации/интеграции для совместимости с актуальными версиями Home Assistant, стандартизации поведения в UI и дальнейшего развития проекта.

Основные цели рефакторинга:

- восстановить работоспособность на современных версиях Home Assistant / HAOS;
- заменить устаревший прямой BLE-сканер на штатный Bluetooth API Home Assistant;
- перевести конфигурацию на config entries;
- привести сенсоры к современному `SensorEntity` API;
- сохранить совместимость со старыми сущностями там, где это возможно;
- подготовить основу для будущего discovery flow, удобного UI настройки, диагностики и поддержки новых устройств Elehant.

Минимальная поддерживаемая и протестированная версия:

- **Home Assistant Core 2026.5.x**

Более старые версии Home Assistant могут работать, но они не являются целевой платформой этого форка.

## Поддерживаемые Устройства

Сейчас интеграция ориентирована на совместимые счетчики воды, продаваемые под маркой Elehant, и поведение исходного проекта:

- однотарифные счетчики с BLE-префиксом адреса `B0:01:02`;
- двухтарифные пакеты с BLE-префиксами адреса `B0:03:02` и `B0:04:02`.

Другие типы счетчиков Elehant, например газовые, пока не заявлены как поддерживаемые до появления реальных устройств или достоверных BLE-сэмплов.

## Поддержка Миграции

Проект поддерживает автоматический импорт старой YAML-конфигурации из исходной версии интеграции.

Старая YAML-конфигурация импортируется в один config entry Home Assistant. YAML используется только как путь миграции, а не как основной формат настройки на будущее.

Интеграция сохраняет старые `unique_id` сущностей:

- сенсоры расхода/объема: `elehant_<id>`;
- сенсоры температуры: `elehant_temp_<id>`;
- двухтарифные сенсоры: `elehant_<meter_id>_1` и `elehant_<meter_id>_2`.

Это предназначено для сохранения существующих entity registry entries, dashboards, автоматизаций и истории recorder в большинстве обычных сценариев миграции.

## Установка

### Установка Через HACS Custom Repository

Пока форк не добавлен в стандартный каталог HACS, его можно установить как custom repository.

1. Сделайте резервную копию конфигурации Home Assistant.
2. Откройте Home Assistant.
3. Перейдите в **HACS**.
4. Откройте меню с тремя точками.
5. Выберите **Custom repositories**.
6. Добавьте URL этого форка.
7. В поле категории выберите **Integration**.
8. Установите интеграцию **Elehant Water**.
9. Перезапустите Home Assistant.

### Ручная Установка

1. Сделайте резервную копию конфигурации Home Assistant.
2. Скопируйте папку:

   ```text
   custom_components/elehant_water
   ```

   в каталог конфигурации Home Assistant:

   ```text
   <config>/custom_components/elehant_water
   ```

3. Перезапустите Home Assistant.

## Первичная Настройка

Перед настройкой убедитесь, что в Home Assistant работает Bluetooth:

- локальный Bluetooth-адаптер виден Home Assistant;
- или настроен Bluetooth Proxy;
- штатная интеграция Bluetooth не сообщает об ошибках.

Если вы устанавливаете интеграцию впервые и у вас нет старой YAML-конфигурации, добавьте ее через:

```text
Settings -> Devices & services -> Add integration -> Elehant Water
```

На текущем этапе удобный discovery flow еще не реализован. После создания config entry счетчики можно настроить через options flow, где временно используется нормализованный JSON `meters`. Более удобный редактор счетчиков и автоматическое обнаружение запланированы как следующие функциональные улучшения.

## Миграция С Исходной Версии

Если у вас уже используется исходная YAML-конфигурация, оставьте ее в `configuration.yaml` на первый запуск после установки этого форка.

Пример старой конфигурации:

```yaml
sensor:
  - platform: elehant_water
    scan_duration: 10
    scan_interval: 30
    measurement: l
    devices:
      - id: 31560
        name: "Hot water bathroom"
      - id: 31561
        name: "Cold water kitchen"
      - id: "31562_1"
        name: "Hot water kitchen tariff 1"
        name_temp: "Kitchen water temperature"
      - id: "31562_2"
        name: "Hot water kitchen tariff 2"
```

Порядок миграции:

1. Установите этот форк через HACS custom repository или вручную.
2. Оставьте старый YAML-блок `sensor: - platform: elehant_water` в `configuration.yaml`.
3. Перезапустите Home Assistant.
4. Интеграция импортирует YAML-конфигурацию в config entry.
5. Откройте **Settings -> Devices & services** и убедитесь, что появилась интеграция **Elehant Water**.
6. Проверьте, что существующие сущности сохранили ожидаемые `entity_id` и `unique_id`.
7. После успешной проверки удалите старый YAML-блок из `configuration.yaml`.
8. Перезапустите Home Assistant еще раз.

Если config entry уже существует, повторный YAML-import не перезапишет его. Это защищает новые настройки от случайной замены старой YAML-конфигурацией.

## Bluetooth

Поддерживаемые счетчики передают данные в non-connectable BLE advertisements. Интеграция использует штатный Bluetooth-стек Home Assistant с режимом `connectable=False`.

В текущей версии в `manifest.json` намеренно не добавлены строгие Bluetooth discovery matchers. Определение пакетов выполняется по широкому legacy-префильтру адреса `B0:*` и разбору manufacturer payload, потому что в рамках подготовки этой версии не использовалась достоверная публичная документация по manufacturer id/prefix Elehant.

## Ограничения

- Автоматическое обнаружение новых счетчиков пока не реализовано.
- `state_class` для долгосрочной статистики намеренно не включен в первом совместимом рефакторинге.
- Поведение availability пока не изменено; интеграция хранит `last_seen` для диагностики.
- Реальные сэмплы двухтарифных пакетов пока отсутствуют; двухтарифный разбор следует логике исходного проекта.
